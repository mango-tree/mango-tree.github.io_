따라서, Django는 “MTV”패턴을 가지고 있다고 말할 수 있다. Model, Template, View.

Python 설치

파이썬 웹 프레임워크이니만큼, 파이썬을 설치하여야 한다.
만약 리눅스나 맥이라면 기본적으로 설치가 되어 있을 것이며, 윈도우라면 Python 홈페이지에서 다운받을 수 있다.

Python 2? 3?

현재 파이썬은 2와 3 두 버전으로 나뉘어 있으며, 둘은 서로 호환이 되지 않는다. 따라서 어떤 버전을 골라야 하는지 고민하게 될 수 있다.
장고는 2와 3 둘 다 사용할 수 있으며, 다른 모듈들이 양 버전을 지원하지 않아 사용을 못하는 경우도 있을 수 있지만 중요한 모듈들은 대부분 2와 3 보두를 지원한다.
필자는 계속 개발이 진행중이며, 인코딩 이슈가 없는 python3를 선호한다. 새로 만들 프로젝트라면 python3를 이용하자.

Django Tutorial 1 - 시작하기

Mar 31, 2017 in Django

Django란?

파이썬으로 만들어진 웹 프레임워크이다. 현재 파이썬 웹 프레임워크 중 가장 높은 인지도를 가지고 있으며, 다양한 기능들로 쉽게 웹사이트를 제작할 수 있다.

Python

앞에서도 설명했듯이 파이썬 기반 프레임워크이다. 파이썬 자체가 쉽고 편한 언어이기 때문에 보다 빠르게 프레임워크에 적응할 수 있다. 또한 많은 파이썬 모듈들을 이용할 수 있다.

MVC 패턴

Model, View, Controller의 약자이다. 웹 애플리케이션의 역할을 크게 세 가지로 나눈 디자인 패턴이며, 간략히 설명하자면:
사용자의 요청(request)를 받아서 - Controller
데이터베이스에서 정보를 읽어서 - Model
사용자에게 페이지를 보여주는(response) - View
역할이라고 볼 수 있다. 각 역할들을 분리하였기 때문에 코드가 뒤죽박죽이 될 가능성(php 날코딩 할 때를 생각해보자)을 확 낮춰준다.

ORM (Object Relational Mapping)

DB를 사용할 때, 쿼리를 일일히 작성하는 것이 번거롭기 때문에 많은 프레임워크에서 나온 기능이다. 보다 쉽게 OOP스럽게 쿼리를 만들 수 있기 때문에 많이들 쓰인다. 예를 들어,

다음과 같은 쿼리를 이렇게 작성할 수 있다.

보다시피 클래스를 이용하여 OOP적으로 DB를 이용하는 것을 볼 수 있다. ORM을 사용한다면 매우 편하게 DB를 관리할 수 있으며, DB를 신경써야 할 부분은 더욱 줄어든다.
다만 쿼리가 매우 복잡해질 경우에는 성능 이슈로 인하여 쿼리를 직접 작성해야 하는 경우가 발생하기도 한다.

Django 공부하기

Django를 공부하기 위한 제일 좋은 방법은 공식 문서를 읽는 것이다. Django 공식 문서는 완벽에 가까운 퀄리티를 가지고 있으며, 모르는 것이 있어 구글링을 할 때도 대부분 첫 번째 글로 검색되어지는 걸 볼 수 있다.
하지만 처음 접하는 입장에서는 무턱대고 공식 문서를 읽으라고 하는 건 진입장벽이 높기 때문에(+영어의 압박), 튜토리얼을 보며 시작하는 것도 나쁘지 않으며, 이 튜토리얼 제작 또한 그런 이유에서 시작하였다.
본 튜토리얼은 Django 1.10 기준으로 진행할 예정이다.
