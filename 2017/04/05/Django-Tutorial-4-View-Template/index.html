Django Tutorial 4 - View

Apr 05, 2017 in Django

이제 웹페이지에 새로운 내용을 채워보자.

새 애플리케이션 생성

장고는 프로젝트 하나에 여러 애플리케이션으로 구성되어 있다. 예를 들어, 웹 사이트 하나(프로젝트)에 여러 기능들과 페이지(유저 정보 관리, 게시판, 결제 등등 - 애플리케이션)로 이루어져 있는 식이다.

새 애플리케이션 만들기

django-admin으로 간단하게 생성할 수 있다.
django-admin startapp <애플리케이션 이름>

메인 디렉토리에 새 디렉토리가 생긴 것을 알 수 있다.

tutorial/settings.py에 들어가서 INSTALLED_APPS에 우리가 만든 애플리케이션을 추가하자

View

Django에서 MVC 패턴 중 “view”는 무엇일까? 특정 URL에 대한 Callback function이다. 이 함수들이 어떤 데이터들이 왔는지 알려주기 때문이다. 그렇다면 “controller”는 무엇일까? Django에서는, 프레임워크 자체라고 볼 수 있다. 프레임워크 단에서 view로 request를 날려주기 때문이다.
따라서, Django는 “MTV”패턴을 가지고 있다고 말할 수 있다. Model, Template, View.
출처: Django 공식 문서

link

장고에서 들어온 요청을 처리하는 부분이다. 웹 페이지의 로직과 알고리즘의 대부분이 이 곳에 있다.
위 공식 문서에서도 볼 수 있듯이, 장고에서 요청을 처리하는 방법은, 요청->라우터(URL 라우팅)->view(->model->view)->template->응답 순이다.
